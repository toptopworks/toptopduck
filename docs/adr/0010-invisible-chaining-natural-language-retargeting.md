# Mental model: chaining is invisible; re-target by natural language

## Decision

物化的 `result_N` 链是**内部实现**，用户表面是一条**线性对话 thread**：每轮 `提问 → 一张表/一张图`。一轮所作用的**当前表**由 LLM 从对话上下文**隐式解析**（默认 = 上一步的中间结果；会话开始 = 源 Dataset），用户通常无需感知 `result_N`。用户可查看**只读步骤历史**（每步 = 提问摘要 + 产出表/图 + 内部标签），并通过**自然语言点名**某步来重定向（如"从去重后的数据，按渠道重新汇总"）；点击历史某步**只是把该引用塞进 prompt 的快捷方式**，不另建 fork/分支 UI 原语。因为派生 + 不可变（ADR-0004），"回溯"天然**长出新枝**而非覆盖旧枝。

## Context

ADR-0001 定位服务**非技术用户**，但 ADR-0003（每步物化）+ ADR-0009（每轮一条 SQL）描述的是 **notebook 链式心智**。若用户必须理解并打理 `result_N`，定位就落空。本 ADR 化解二者冲突：把链藏起来。

## Why

1. **非技术用户不能被要求管理 `result_N` 链**——否则 ADR-0001 的定位破产；线性 thread 才是他们的心智。
2. **自然语言重定向复用已有 LLM 解析能力，不引入额外 UI 原语**（KISS）；点击仅是注入引用的快捷方式，同一回路。
3. **回溯长出新枝是 ADR-0004 派生 + 不可变的自然结果**，无需额外设计——分支是免费的副作用。
4. 历史视图给用户**用自然语言精准指代**所需的信息（标签 + 摘要），是隐性链的纠错抓手。

## Considered options

- **纯点击式显式控制（notebook 风格）**：更可控但更重，且与非技术用户定位相悖。**否决**。
- **只读历史、不支持重定向**：最省，但用户改错粒度只能重开会话，链式迭代体验断裂。**否决**。

## Consequences

- `result_N` 会**分支膨胀**（同一中间结果被多个后续引用）→ 必须配套生命周期 / GC / 上限（见后续 ADR）。
- "当前表"解析**失败**（LLM 把"那个"指错）时，用户须能纠偏；历史视图的标签是纠偏入口。
- 历史视图每步须渲染该轮的 `assumption`（0009 契约字段）为可纠偏旁注——它是隐式当前表解析出错时的纠错入口。
- **历史视图是必要 UI，非可选项**——它是不可见链唯一的纠错与回溯接口。
- 这条 thread 的呈现即 ADR-0009 说的"notebook cell"，但它是**呈现层**，不是用户要打理的对象。
- 历史视图的标签/摘要须**同时喂给 LLM**（与给用户看同源），否则点名重定向老步会断——见 ADR-0013 窗口外载荷含摘要。
- **被 ADR-0039 校准**：历史视图步标签 / 远期指代句柄 = **用户原话（有界截断），非 LLM 生成的标题**；同源喂 LLM 与给用户看（本 ADR「同源」不变）。
- **被 ADR-0048 延伸**：「点击历史步塞引用 + 不开新原语」延伸至**澄清应答与假设纠偏**——两者都走「下一条自然语言轮次」，点 `assumption` 旁注 = 聚焦 QuestionBar（非新原语）。见 ADR-0048。
