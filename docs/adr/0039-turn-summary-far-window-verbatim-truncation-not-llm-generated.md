# Turn summary in the far window: verbatim user question (bounded truncation), never LLM-generated

## Decision

「提问摘要」（远期对话窗口外那一行、历史视图的步标签、LLM 远期重定向的映射句柄——ADR-0010/0013/0023）= **用户原话的有界截断（首句 / 截断）**，**非 LLM 再生成**。窗口内（近 N=20 轮）仍发**全量原话**（ADR-0023，不变）；仅窗口外用截断。同一份原话（截断）同时是用户可见的步标签与喂 LLM 的远期指代句柄（ADR-0010「同源」）。

## Context

ADR-0023 窗口外载荷 = `提问摘要 + result_N 名`，并要求「摘要忠实」；ADR-0010 说这份摘要同时喂 LLM 与给用户看；ADR-0013 亦用「提问摘要」作远期 `result_N` 的指代。但**这行由谁产生从未定义**——用户原话截断，还是 LLM 每轮额外生成？整个窗口外远期重定向机制压在此物上，却无 ADR 钉死其本质。

## Why

1. **天然忠实 + 零成本**：= 用户原话直接满足「须忠实」（无软保证负担），零额外 LLM 调用（KISS）。
2. **原话含真实指代词**：「把**那个去重表**重算」里的指代正是远期重定向所需原汁信号；LLM 压缩反而可能洗掉。
3. **无新契约 / 无新失败路径**：不扩展 LLM 输出契约（无 summary 字段）、无「摘要生成失败」分支。
4. **窗口内已全量**：截断只发生在远期窗口外；近 20 轮恒为全量原话，远期重定向低频，截断有损可接受。

## Considered options

- **LLM 每轮生成一行摘要**：压缩干净，但多一次生成（成本 / 延迟）、「忠实」成软保证、须扩展输出契约 + 处理摘要生成失败。**否决（KISS + 忠实优先）**。

## Consequences

- **校准 ADR-0023**：「提问摘要」= 用户原话有界截断、非 LLM 生成；窗口内全量不变。0023 Consequences 已追加指针。
- **校准 ADR-0010**：历史视图步标签 = 用户原话（截断），同时喂 LLM；非生成式标题。0010 Consequences 已追加指针。
- **校准 ADR-0013**：远期 `result_N` 的指代句柄 = 该轮提问原话（截断）。0013 Consequences 已追加指针。
- **CONTEXT 已更新**：「提问」条目补远期 / 历史指代语义。
- 实现侧：窗口外载荷取提问原话首句 / 截断（有界长度）；无 LLM 摘要生成步骤；截断边界为 impl 调参。
