# Source lifecycle ops serialized with turns; the event is first-class in thread but not a turn — excluded from the LLM turn-window

## Decision

源生命周期操作（加源 ADR-0022 / 换源 ADR-0025 / 删源）是**会突变工作集 / recipe 结构的操作**，与"创作新轮"在运行态**互斥**（会话内线性化）：

1. **互斥口径 = 执行窗口**：一轮从提问提交到**达终态**（ADR-0028 的 A/B/C/D outcome）之间为"在飞"；在飞期间**源管理 UI 全禁用**。达终态后的轮间间隙（**含消歧 B 轮等用户回答时**——消歧已达 B 终态、非在飞）源管理允许，与任何两轮之间同。粒度 = **会话内**，多个不同 `.duck` 会话并行互不影响（与 ADR-0027 内存隔离、ADR-0035 同文件单写者正交）。

2. **加源不开例外**：加源虽是纯追加、不级联失效，仍纳入互斥——以保崩溃窗口单一（见下），规则一刀切（KISS）。加源 / 查询并发留 v2 扩展点（呼应 ADR-0021 把并发查询留 v2）。

3. **写盘 = 轮终态写的机械镜像**：源操作达终态（copy-in 完成 + stale 标记完成）**立即**整文件重写落盘（临时文件 + rename，ADR-0034）；进行中不写；崩溃窗口 = 该源操作进行中（rename 前），崩了原文件完整、resume 见源操作前态。写盘前同样走 ADR-0035 的文件 hash 比对（外部变更检测），命中则挂起写、走 0035 三选一。源操作不特殊——它是 0034"每突变达终态即原子整文件重写"的又一实例，**无新写盘契约**。

4. **源生命周期事件 = 对话 thread 的一等公民，但非轮次**：源操作在 recipe 的对话历史（展示部分，ADR-0034 全量纯追加）中**追加一条可见事件**，占**时序位置**、恒可见；但它**无提问、无 outcome，不是轮次**（ADR-0028），故：
   - **不进 LLM 轮次窗口**（ADR-0023）——窗口按轮次裁剪，源事件非轮、天然不进该子集；
   - **不占 N=20 槽、不动 result_N 编号**（0028 三不变量是**轮次**的，源事件不在其管辖——非"可见不占窗"第三态例外，是"非轮天然不进轮次窗口"）；
   - 其携带的工作集变更已通过"全部源全 schema 始终全量发送（0023）+ stale 派生不进 LLM 工作集（0013）"传递给 LLM，故源事件对 LLM 是冗余、不占窗无损语义；
   - 它是 stale 派生"因何失效"标注（ADR-0025）的**锚点**——每个 stale result 溯源到失效它的那条源事件。

## Context

ADR-0034 把"轮终态写"的崩溃窗口钉死（= 在飞那轮），但对"源生命周期操作的写盘"只一句"原子落 recipe"——**何时写、崩溃窗口多大、与在飞轮如何排序**未钉死；ADR-0035 的"单写者"管的是**文件级**并发（同 .duck 多开 / 外部编辑），不是**进程内一个会话**里"源操作 vs 在飞轮"的**逻辑**并发。又：0034 说源操作"追加对话历史一条可见事件"，但 ADR-0028 把"轮次"钉死为"提问 + outcome"并立三不变量（恒可见 / 恒占 N=20 窗口 / 不动编号）——这条"可见事件"既无提问也无 outcome，**如何嵌进轮次体系、占不占窗口槽**从未定义。是持久化主题（0034–0039）收口后暴露的、源变更维度的未闭合依赖。

## Why

1. **互斥直接消除最毒交错**：换源 / 删源级联 soft 失效派生（0025）。若允许与在飞轮交错——在飞轮 T 的 SQL `FROM result_5`，而 result_5 依赖的源被换 → result_5 立即 stale → T 达终态时引用刚被裁出可重放链的死轮，"T 落盘时依赖死轮"recipe 无法干净表达。互斥让该状态不可达。
2. **崩溃窗口单一**：互斥 → recipe 任何时刻最多一个突变在途，崩溃窗口恒为"那个未写突变"，与 0034 整文件重写同构。加源例外会引入"查询在飞 + 加源在途"两突变并发，须新立进程内并发写契约（0035 单写者未覆盖同会话两操作并发写），为窄收益引大复杂度。
3. **契合 0021 线性心智**：会话是单用户线性 thread（0010）；0021 已禁查询并发，源操作是更重突变，更该排队。
4. **源事件不进 LLM 窗口无损语义**：其对 LLM 的价值是"工作集变了"，而该信息已被"源全量发送 + stale 不进工作集"覆盖；强行占 N=20 槽只挤掉语义相关轮次槽位。对话历史（全量真相）与 LLM 窗口（按轮次派生子集）解耦——正是 ADR-0036 内容边界"对话窗口从全量派生"的落地。

## Considered options

- **源操作与在飞轮允许交错**：换源 / 删源交错出"T 引用刚失效 result_N"的不可表达态（见 Why 1）；崩溃窗口与进程内并发写契约复杂化。**否决**。
- **加源开例外（仅换源 / 删源互斥）**：崩溃窗口双重化 + 须立同会话并发写契约；加源秒级、收益面窄。**否决（v1）**。
- **源事件占 N=20 窗口槽（与轮平级）**：对 LLM 冗余信号挤占语义槽位；违背"窗口按轮次"。**否决**。
- **源事件作相邻轮的标注、不占时序位置**：换源发生在两轮之间、挂哪轮语义模糊；无法表达"换源时刻"，且 stale 锚点无独立位。**否决**。

## Consequences

- **延伸 ADR-0034**：0034"源生命周期操作原子落 recipe"被钉死为——会话内与轮互斥、达终态立即整文件重写、崩溃窗口 = 该操作进行中、追加一条**源生命周期事件**（非轮）进对话历史。0034 Consequences 已追加指针。
- **澄清 ADR-0028 边界**：0028 三不变量（恒可见 / 恒占窗 / 不动编号）是**轮次**的；源生命周期事件非轮次，不在其管辖——"不占 N=20 槽"非"可见不占窗"第三态例外，是"非轮天然不进轮次窗口"。非改 0028 文本，是明确其适用边界。0028 Consequences 已追加指针。
- **延伸 ADR-0021**：禁用口径从"查询进行时禁输入框"扩为"执行窗口（提问已提交、outcome 未达）期间禁源管理 UI"；消歧 B 轮已达终态、其间隙源管理允许。0021 Consequences 已追加指针。
- **澄清 ADR-0023**：N=20 窗口"按轮次"明确排除源生命周期事件（非轮）；对话历史全量含源事件、窗口从中只取轮次。0023 Consequences 已追加指针。
- **正交 ADR-0035**：本 ADR 的"会话内逻辑互斥"与 0035"文件级单写者 / 外部变更检测"是两个正交并发层——前者管同会话内操作时序，后者管同文件多实例 / 外部编辑。不互校准、互补。
- **CONTEXT 已更新**：新增「源生命周期事件（Source Lifecycle Event）」术语；「轮次」条目追加交叉引用（对话 thread 含此类非轮事件）。
- 实现侧：执行窗口内源管理 UI 禁用、源操作达终态整文件重写、对话历史序列化含源事件（操作类型 + 源引用名 + 时序位）、stale result 记"被哪条源事件失效"以溯源。
- **未闭合依赖（下一支）**：换源级联失效后，失效轮的 SQL 被裁出可重放链（0034）但保留于对话历史展示；若用户换源后"用新源重算刚才那个"——SQL 在对话历史却不在 LLM 窗口，LLM 如何取到？留待下一 ADR。
