# Visualization triggering & degradation: LLM decides viz (NL override, no button); degraded viz must be disclosed

## Decision

1. **LLM 自决 viz**：每轮是否带 viz、带哪种（白名单内，ADR-0016）由 LLM 按查询意图 + 结果形状自决，system prompt 引导（趋势 / 对比 / 分布类意图 → 图；查明细 / 查值 → 表）。**默认表格**，明确视觉意图才图。用户可**自然语言覆盖**（「画个柱状图」/「就显示表格」）。**不设 viz 切换按钮。**

2. **viz 退化须披露**：凡 emitted 的 viz 渲染失败（Vega-Lite 校验失败、重试后仍失败，ADR-0016 退化表格），UI 必须**明示「图表无法渲染，已显示表格」**——诚实不变量，不论用户是否显式要过图。

## Context

ADR-0009 契约 `viz?` 字段可选、ADR-0016 定 viz 格式（Vega-Lite + 白名单）与**畸形恢复**（校验失败 → 重试 → 退化表格），却都没钉：**谁决定一轮是否带 viz**（LLM / 规则 / 用户）？以及 viz **退化时是否须披露**（静默退化 = 用户显式要的图凭空变表格、却不被告知）。是诚实脊柱（0011/0017/0030）下、viz 维度的未闭合依赖。

## Why

1. **复用「LLM 解析意图、用户 NL 纠偏、藏控件」哲学**（0010 隐式当前表、0022 自然语言切源、0018 假设标注）：viz 触发是同一套模式的又一维度——LLM 默认决、用户 NL 覆盖、不设 viz 按钮（与 0022 否决「切换当前表按钮」同源，不戳破 0010 心智）。
2. **viz 是展示层、非数据层**——错图 / 缺图不污染 `result_N`、不误导数据（表格始终在）；viz 判断失误代价 = 「次优图表、用户一句话改」，远低于 SQL 判断失误。「LLM 决 viz」可接受门槛低于「LLM 决 SQL」，放手决、错了低危易纠。
3. **静默退化 = 沉默的误导**：用户显式要图却看到表，不说明则或以为「图就是这表」（困惑）、或以为请求没生效（重问）；一律披露省去「区分用户要 vs LLM 自决」的分支（KISS），退化本罕见 + 旁注轻量。
4. **退化披露与 0010 历史视图一致**：退化轮在历史亦标注「图表渲染失败 → 表格」，保住可检视 / 可纠偏信任。

## Considered options

- **规则触发 viz（如聚合才画）**：僵，漏掉显然该画但规则不触发的情形；0016 已否决「纯表格太瘸」。**否决**。
- **用户显式要才画 viz**：摩擦，违背非技术用户（0001）「问一句、出洞察」流。**否决**。
- **设 viz 切换按钮**：戳破 0010 藏控件心智，与 0022 否决「切表按钮」相悖。**否决**。
- **静默退化（不披露）**：沉默误导，违背诚实脊柱。**否决**。
- **仅用户显式要图时披露退化、LLM 自决图不披露**：分情况判断、徒增分支；一律披露更简且诚实。**否决**。

## Consequences

- **校准 ADR-0009**：`viz?` 字段填充语义明确为「LLM 按查询意图自决是否 / 填哪种（白名单内）；用户可 NL 覆盖；不设 viz 按钮」。0009 Consequences 已追加校准指针。
- **校准 ADR-0016**：补 viz 触发（LLM 自决 + NL 覆盖 + 无按钮）与退化披露（emitted viz 渲染失败 → 明示「图表无法渲染，已显示表格」，不论是否显式要过）。0016 Consequences 已追加校准指针。
- 实现侧：system prompt 引导保守默认（默认表、视觉意图才图）+ 白名单（0016）；前端渲染退化旁注（非阻断）；ADR-0010 历史视图渲染退化标注。
- viz 退化不改变 0028 outcome 分类——SQL 有效、`result_N` 已物化 → 仍是 outcome A（结果轮），仅 viz 维度退化（presentation 层），不走 failure-outcome / void。
