# Sample rows to LLM: first-3-rows, fixed at copy-in, windowed Datasets only

## Decision

每个**窗口内** Dataset 发给 LLM 的样本 = **首 3 行**（按 copy-in 后的物理顺序），在 **copy-in 时固定一次、会话内永不变**。源 Dataset 永远全量发（含样本，源不裁剪）；窗口内 `result_N`（近 N=20 轮，ADR-0023）发首 3 行样本；**窗口外 `result_N` 不发样本**（只发表名+列名+摘要，ADR-0013/0023）。用户可按数据集关/减、按列脱敏（ADR-0011）。

## Context

ADR-0011 定"默认发 3 行真样本"但未定：哪 3 行？何时固定（每轮随机 vs 一次固定）？窗口外 `result_N` 还发不发？样本同时服务 SQL 质量、类型推断（ADR-0020）、隐私披露（0011）三方，选取策略必须钉死。

## Why

1. **固定一次 = 稳定 + 可复现 + 隐私确定**：每次 prompt 发同一组样本，LLM 行为可预测、SQL 可复现；隐私披露（0011）是"确定的 3 行"而非每次随机。与 ADR-0012 源快照不可变一致——样本是快照的一部分。
2. **首 3 行 = KISS 且够用**：样本用途是让 LLM **写对 SQL**（看类型/格式/单位/null 模式），这些通常行间一致，首 3 行足以暴露；分布统计（corr/quantile）是 SQL 算出来的，不靠样本。脏表头/异常首行由 ADR-0015 规整 + 0020 样本兜底处理。
3. **否决随机固定种子**：代表性略好但增加复杂度 + 隐私面（随机抽中特殊敏感行），v1 不值（YAGNI）。
4. **窗口外不发样本**：与 ADR-0013/0023 的远期裁剪一致——窗口外 `result_N` 只留摘要保重定向能力，样本是 token 大头，窗口外省掉。

## Considered options

- **随机固定种子**：代表性略好，但复杂 + 隐私面 + 稳定性弱于固定首行。**否决（v1）**。
- **分层/去重采样**：最稳但 v1 过度（YAGNI）。**否决（v1）**。
- **每轮重新随机**：分布暴露好，但 SQL 不可复现、隐私不确定。**否决**。
- **窗口外 result_N 也发样本**：token 失控，违背 0013/0023 裁剪。**否决**。

## Consequences

- **校准 ADR-0011**："每 Dataset 3 行"精确化为"窗口内 Dataset 发首 3 行，copy-in 时固定一次；窗口外 `result_N` 不发样本"；0011 Consequences 已追加校准指针。其余（按数据集/按列脱敏控制）不变。
- 样本选取为首 3 行、copy-in 固定，是 prompt 契约的确定部分；排序数据首 3 行不代表整体是已知代价（样本服务"写 SQL"非"做统计"）。
- 窗口内外的样本边界与 ADR-0023 轮次窗口对齐：源全量 + 近 N=20 轮含样本 + 远期无样本。
- **被 ADR-0032 校准**：本 ADR 发的 schema 载荷中「类型」部分 = 物理类型原样 + 嵌套全展开（ADR-0032）；首 3 行样本补值、不替代嵌套结构（可选字段 / 深嵌套难从值反推），两者互补不互代。
