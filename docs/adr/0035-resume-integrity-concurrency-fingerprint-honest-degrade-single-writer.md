# Resume & concurrency failure contract: per-source content fingerprint, honest degradation + re-link, in-process single-writer, external change detection

## Decision

1. **resume 完整性校验**（接 ADR-0034 的 resume 管线，重放前对每个源校验**存在性 + 内容指纹**——recipe 已记每源 hash）：
   - **都在且指纹符** → 静默进入重放。
   - **源消失**（删 / 移 / 改名）→ 阻断，诚实报「找不到 X」，提供 **re-link**：用户指到搬走后的新路径，recipe 只更新路径即继续——正因 recipe 不存数据（0034），re-link 才有意义。
   - **源漂移**（同路径但指纹不符）→ 诚实报「该源自上次使用后已变化，重算结果可能不同 / SQL 可能报错」，让用户决定重建或中止；**绝不**静默用新数据当旧数据算。
   - **多源部分失效**（ADR-0022）→ 受影响源逐个走上面，未受影响源照常重建（共享命名空间下按源隔离处理）。
   - **active 源被放弃时不自动回落**（接 ADR-0037：active 指针 = 引用名）：若用户对某失效源放弃 re-link（= 删源，ADR-0034；其派生级联 soft 失效，ADR-0025/0013），而该源正是 active-dataset → **不自动回落**到剩余源，而是**阻断 + 显式让用户选**「在剩余源中指定一个继续 / 中止 resume」。一个源都不剩 → resume 到无 active 的空工作集并提示上传。active 是用户分析焦点，静默跳源是隐式状态变更——与诚实脊柱（0011/0017/0030）一致，宁可多一次打断也不替用户猜。

2. **重放中途某轮 SQL 失败**（如漂移后列被删）→ 该轮按 ADR-0028 failure outcome + ADR-0030 失败披露呈现，工作集物化到断点、对话停在断点，用户能看到**第几步断了**。

3. **并发：进程内单写者 + 外部变更检测**：
   - **进程内单写者**——同一 app 进程拒绝重复打开同一 .duck（in-process 追踪已开文件，零 OS 锁、无遗留锁）。
   - **跨进程 / 外部变更检测**——每次自动写盘前（接 0034）比对文件当前 hash vs 上次写的 hash：变了 → **挂起该轮自动写、诚实提示**「该 .duck 在磁盘上已变化：重载（丢内存未写轮）/ 保留我的（覆盖）/ 另存为新文件」，**绝不静默 clobber**。
   - app 关着时外部编辑 .duck 是可移植文档特性（0034），打开即按新 recipe 走。

## Context

0034 让 .duck 可移植 + 每轮自动写，必然产生两类新失效：**源失效**（recipe 只存路径引用、源数据不进 .duck）与**文件并发**（同 .duck 多开 / 外部编辑）。ADR-0017 钉了能力边界诚实拒绝，但「数据时效边界」（拿陈旧 / 错位数据算结果）未覆盖；ADR-0028 钉了 outcome 契约但未涉 resume 重放中的失败呈现；0027 钉了会话间**内存**隔离但未涉同一 .duck 的**文件级**并发。是诚实脊柱（0011/0017/0030）下、持久化引入的新失效维度的未闭合依赖。

## Why

1. **内容指纹是把"静默漂移"变"可见漂移"的唯一机制**——hash 必须从首次保存就记（否则老文件无指纹可比），故 hard-to-reverse，必须现在做；save/resume 各一次流式 hash，对大文件可接受。
2. **诚实降级把 0017 能力边界诚实拒绝延伸到数据时效边界**——不诚实拿陈旧 / 错位数据算出结果，比直接拒绝更糟；与 0030「截断 / 空须披露」同源。
3. **re-link 因 recipe 不存数据才有意义**（0034）——失效只需更新路径，不需重传数据。
4. **进程内单写者零成本解决最高频并发**（同 app 两窗口开同文件）；OS 咨询锁在 Windows 不可靠（本应用 win32/mac/Linux 桌面）且要遗留锁清理子系统，单用户桌面"两独立进程开同文件"罕见——YAGNI，变更检测照样警告。
5. **变更检测拒绝静默 clobber**，与数据时效诚实一致；外部编辑本就是可移植文档的特性，检测而非禁止。

## Considered options

- **静默尽力（不校验，读到啥用啥，炸了才报）**：静默漂移 = 沉默误导，违背诚实脊柱。**否决**。
- **OS 咨询式文件锁**：Windows 不可靠 + 遗留锁清理子系统；单用户桌面两独立进程开同文件罕见。**否决（v1）**。
- **last-write-wins 无检测**：两个 live 实例互相 clobber，真实数据丢失。**否决**。
- **检测到外部变化后静默保留我的（覆盖）**：静默 clobber 他方改动，不诚实。**否决**。

## Consequences

- **延伸 ADR-0017**：诚实拒绝从「能力边界」扩到「数据时效边界」（resume 漂移 / 缺失诚实拦截）——非改 0017 文本，是其哲学在持久化层的映射。
- 重放失败呈现复用 0028 outcome + 0030 披露，**无新 outcome 类型**；多源部分失效复用 0022 共享命名空间隔离。
- 实现侧：源 hash 计算（save/resume 各一次流式）、resume 校验 + re-link UI（含 active 源被放弃时的显式选源 / 中止）、进程内已开文件追踪、写盘前 hash 比对 + 冲突提示。
- **延伸 ADR-0037**：active-dataset 指针（引用名）在其 backing 源被放弃时**不自动回落**——阻断 + 显式选源（诚实脊柱），与 0037「指针稳定」正交；删源级联失效复用 0025/0013。
