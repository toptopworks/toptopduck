# Result materialization edge cases: empty (0-row) results materialize normally; display truncation must be disclosed

## Decision

两条精确化 ADR-0003/0024 的物化规则：

1. **空结果（0 行）正常物化**：一条返回 0 行的 SQL 仍物化为一个正常 `result_N`——一张 0 行、带 SELECT 投影 schema 的空物理表，消耗编号（ADR-0022）、进工作集、可被后续 SQL 引用。**不特判为「无结果」**。

2. **展示截断须披露（诚实不变量）**：当 `result_N` 行数 > 展示上限（默认 1 万，ADR-0005/0024），UI 必须**显式披露**——明示总行数 +「显示前 N / 共 M 行；导出与后续 SQL 见全量」。**不得静默以截断结果冒充完整结果。**

## Context

ADR-0003「每步自动物化」与 ADR-0024「物化为物理表、展示分页 `LIMIT 10000`、物理表存全量」都假设「有结果」，却未钉：**0 行结果**是否物化（还是特判「无结果」）？以及展示截断（用户只见 1 万、表存全量）**是否须披露**——静默截断会让用户把部分当全部、下错误结论，是诚实脊柱（ADR-0011/0017）下的未闭合依赖。

## Why

1. **0 行机械上自然 + 是有效答案**：`CREATE TABLE result_N AS <sql>`（ADR-0024）在 0 行时本就产出空表（带投影列结构），无需特判；「符合条件的没有」是正当分析结论，特判 = 把答案降级成异常。
2. **空结果可链式**：下游可 `SELECT COUNT(*) FROM result_N` = 0、或做差集/反查（`WHERE NOT EXISTS (SELECT 1 FROM result_N)`）；不物化就断链。空表列存压缩近乎零成本，M=100 GC（ADR-0013）兜底噪音。
3. **静默截断 = 沉默的误导**，正是 ADR-0011/0017 要消灭的「自信的错误」；披露让用户知「看到的非全部、但全量在、可导出/可链式」。
4. **截断是展示层、非数据层**：物化表存全量（ADR-0024）、后续 SQL/导出见全量；披露只发生在「给用户看那一眼」，不改 prompt 契约（LLM 拿 schema + 首 3 行样本，ADR-0026，不拿展示行/总行数）。

## Considered options

- **0 行特判为「无结果」（不物化、不占编号）**：破坏「一条 SQL → 一个 result_N」不变量、引入第 5 种 outcome（ADR-0028）、断链式。**否决**。
- **静默展示截断（只分页、不披露总行数）**：沉默误导，违背诚实脊柱。**否决**。
- **披露仅作 UX 常识、不进 ADR**：钉的是诚实不变量（防未来「为干净」去掉行数提示而静默误导），非渲染细节；不进 ADR 则无约束力。**否决（不进 ADR）**。

## Consequences

- **校准 ADR-0003**：「每步自动物化」明确含 0 行结果——空结果物化为正常 `result_N`（0 行 + 投影 schema）。0003 Consequences 已追加校准指针。
- **校准 ADR-0024**：物化物理表明确含 0 行（空物理表）；「展示分页」补「截断须披露」诚实不变量（明示总行数 + 截断状态）。0024 Consequences 已追加校准指针。
- **校准 ADR-0005**：「1 万行展示 + 全量可导出」补「展示行 < 总行数时须披露」——静默截断为禁止行为。0005 Consequences 已追加校准指针。
- 渲染细节（badge 形态、分页交互）留 impl；架构层只钉「空结果正常物化 + 截断必披露」两条不变量。
