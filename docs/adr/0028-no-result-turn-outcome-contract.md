# No-result turns: unified "turn outcome" contract — always visible, windowed, single numbering; one retry loop for failures

## Decision

一次**轮次**（用户提问 + 一个 outcome）无论 outcome 是否产出 `result_N`，**恒在对话 thread / 历史视图中可见**；产不产中间结果只决定 outcome 类型。统一四类 outcome，共享三条不变量，仅在「是否可恢复」上分档：

**四类 outcome**：
- **A 结果轮**：产 `result_N`（ADR-0003/0009）。
- **B 文本态轮**：LLM 主动不产 SQL——澄清（ADR-0018）/越界拒绝（ADR-0017）。
- **C 失败轮**：重试回路耗尽——畸形输出（契约违反）+ 执行逻辑错（schema 错 / 运行时逻辑错如除零、类型转换、溢出）。
- **D 取消轮**：abort——用户取消 / 资源上限 / 语句超时（ADR-0021）。

**三条共享不变量**：
1. **恒可见**：B/C/D 不产 `result_N` 仍作为一条时序条目存在（守住 ADR-0010 线性 thread 心智与纠错回路）。
2. **恒占 N=20 窗口一槽**（校准 ADR-0023）。窗口内/外载荷按 outcome 类型：

   | outcome | 窗口内（近 N=20） | 窗口外（一行摘要） |
   |---|---|---|
   | A 结果轮 | 提问 + SQL + result_N 全 schema（含样本） | 提问摘要 + result_N 名 |
   | B 文本态轮 | 提问 + 文本响应（澄清/拒绝） | 提问摘要 +「澄清/拒绝了 X」 |
   | C 失败轮 | 提问 + 失败原因 | 提问摘要 +「失败：{原因}」 |
   | D 取消轮 | 提问 +「已取消」标注 | 提问摘要 +「已取消」 |
3. **不动 `result_N` 编号**（校准 ADR-0021）：编号单调递增、允许空洞、永不复用（ADR-0022）；只有 A 推进它。轮次本身仅有**时序位置**，无独立步号计数器、不向用户展示步号。

**恢复档**（C/D）：
- **C 失败轮 = 单预算重试回路**（校准 ADR-0005/0009）：畸形输出 + 执行逻辑错**共用每轮一个总重试预算**（默认 2 次），回喂失败（DuckDB 错 / 契约错），耗尽则如实告知、落为 failure-outcome。**资源上限/超时（D）不进回路**（同 SQL 必重撞同墙）。重试对用户不可见——一次提问 = 一条时序条目 = 一个 outcome。
- 执行逻辑错的改写若**实质改变语义**（除零→过滤、聚合口径变），须走 `assumption`（ADR-0009）显式标注；纯 schema/对象名修复不改变语义，无需标注（复用 ADR-0018「做了+标注 > 硬猜」哲学）。

## Context

ADR-0009 契约 `{ sql, viz?, assumption? } | 文本态` 只覆盖了 A 与 B 两类「有明确产物」的轮次，但「**没产出**」的轮次散落在 ADR-0017（拒绝）、0018（澄清）、0021（取消）、0005 L4（schema 错重试耗尽）四条 ADR，各自定义、从未统一；且 **LLM 输出违反契约**（畸形 JSON / 既非澄清也非 SQL）这一路径**完全无 ADR 覆盖**。叠加：这些轮次算不算可见、进不进 ADR-0023 的 N=20 窗口、占不占 ADR-0022 编号、历史视图如何彼此区分——每条 ADR 各说各话（0021「不计步序」、0010 历史视图默认「每步有产出表/图」），是 ADR-0010 线性 thread 心智下的未闭合依赖。

## Why

1. **统一「轮次恒可见」守住 0010 线性 thread 心智**：用户打了一句话，thread 不能凭空消失——否则破坏线性对话承诺、断掉纠错回路。把 0021 为取消单开的「历史可见」口子提升为 B/C/D 的通用不变量，更一致。
2. **可见 / 占窗 / 编号 三维度解耦**：避免「不可见=不计数」暗箱。B/C/D 全部可见、全部占窗——最近意图含失败/取消，是写对下一轮 SQL 的高价值上下文（尤以执行失败为甚，防 LLM 重蹈覆辙），但全不动编号（编号只服务血缘，与可见性正交）。
3. **C 单回路统一 path 4/5**：畸形与执行失败语义同构（都是「非用户、非越界原因没产出可用结果」），合一回路 = KISS/DRY；共享单预算防 malformed→execution→malformed 重试风暴；重试对用户不可见保住「一次提问=一条条目」原子性。
4. **执行失败两档（可重试 vs abort）有原则**：资源上限/超时重试必撞同墙 → abort；逻辑错可改写 → 重试，配 `assumption` 标注语义性改写保透明。否决第三档（立即告知运行时错）——逼系统判「修复改没改语义」是易错元判断，YAGNI。

## Considered options

- **B/C/D 从 thread 抹掉**：破坏 0010 线性心智、断纠错回路。**否决**。
- **cancel 可见但不占 N=20 槽**：引入「可见却不计窗」第三态，多一维例外要维护。**否决**。
- **畸形输出立即硬抛、不重试**：浪费一轮在 LLM 瞬态抖动（截断常见）上，对非技术用户（ADR-0001）是莫名报错。**否决**。
- **宽容修复（正则从散文抠 SQL）**：把 0005 已否决的「解析 LLM 输出」脆弱性从 SQL 层搬到 JSON 层，换汤不换药。**否决**。
- **畸形与执行失败各开独立重试预算**：更防连累，但单轮内两类连续命中极罕见、且兜底安全；按类型分桶留 v2。**否决（v1）**。
- **运行时/逻辑错立即如实告知、不重试**：放弃 LLM 加 `NULLIF`/`CASE` 自动修的机会；且判「改没改语义」的系统元判断易错。**否决**。

## Consequences

- **校准 ADR-0009**：契约的「失败路径」明确为——畸形输出（契约违反）走 C 失败轮单预算重试回路，耗尽落 failure-outcome；0009「单查询失败回喂重试 1–2 次」被吸收进统一单预算。0009 Consequences 已追加校准指针。
- **校准 ADR-0005**：L4「schema 错重试 1–2 次」扩展为「schema 错 + 运行时/逻辑错 + 畸形输出共用单预算重试回路」；L3 资源上限/超时仍走 ADR-0021 abort，不进回路。0005 Consequences 已追加校准指针。
- **校准 ADR-0021**：「cancel 不计步序」精确化为「步序 = `result_N` 编号（单一轴，ADR-0022）；cancel 轮（D）可见、占 N=20 槽、但不动编号」；轮次仅有时序位置、无独立步号计数器、不向用户展示步号。0021 Consequences 已追加校准指针。
- **校准 ADR-0023**：N=20 窗口从「仅枚举结果轮 + 文本态轮」扩展为「全部四类 outcome 均占槽」，窗口外载荷按 outcome 类型（C→「失败：原因」、D→「已取消」）。0023 Consequences 已追加校准指针。
- **CONTEXT 已更新**：新增「轮次（Turn）」术语（恒可见、outcome 类型化、可见/占窗/编号三维度独立）；「提问」收紧为「轮次的输入」，不再承诺每次必出结果。
- 历史/线程视图须能按 outcome 类型区分渲染（结果/文本/失败/取消）——渲染细节为 UX 子决策，契约层只需保证四类可区分（Decision 载荷表已给）。连续同类非结果轮可折叠/弱化以防噪音，但**条目不可不存在**。
- 默认重试预算（2 次）、N=20、M=100 均可调。
