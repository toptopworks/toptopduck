# 澄清应答与假设纠偏：走「下一条自然语言轮次」，不开新 UI 原语

## Decision

两类「用户修正 LLM 理解」的交互——**澄清应答**（B 轮，ADR-0018）与**假设纠偏**（A 轮 `assumption`，ADR-0010/0009）——都走**同一条回路：用户的下一条自然语言轮次**，**不引入任何新 UI 原语**（无 quick-reply chips、无 inline-edit、无结构化选项）：

**（1）澄清应答（B 轮，rare）**
- LLM 返澄清文本（B 轮、无 SQL，ADR-0018 窄门槛）→ rail 一张 `○` 卡（0047）+ workspace 文本载荷卡显示澄清问题（0045 / Q8 路由）。
- 用户**直接在下一条 QuestionBar 输入回答**——无 quick-reply chips、无选项按钮、无预填魔法内容（最多占位提示）。
- LLM 从 ADR-0028 的 N=20 窗口读到澄清轮 + 回答轮，据此生成下一轮 SQL。

**（2）假设纠偏（A 轮，common）**
- A 轮 `assumption`（ADR-0009 契约字段）渲染为 workspace 结果区的**可纠偏旁注**（ADR-0010 / 0016）。
- **点旁注 = 聚焦 QuestionBar + 占位「纠正此假设…」**；用户输入纠偏 → 作**新轮**提交。
- 纠偏产**新 `result_N`**（编号不重用，ADR-0022）；**原轮（误读的）不动**——守 ADR-0010「回溯长出新枝非覆盖」+ ADR-0041「不复活」。

**核心原则**：澄清 / 纠偏复用既有「下一条自然语言轮次」通道，与 ADR-0010「自然语言重定向」同一回路——**不开新原语**。

## Context

ADR-0010 立「自然语言重定向复用已有 LLM 解析能力，不引入额外 UI 原语」，并把历史步的 `assumption` 旁注定为「隐式当前表解析出错的纠错入口」；ADR-0018 把歧义分两路——默认「做了 + 标注」（A 轮 assumption）、窄门槛「澄清文本」（B 轮）；ADR-0009 契约定 `assumption` 为可选字段、文本态承澄清 / 拒绝。但**澄清怎么答、假设怎么纠**的交互 UX 从未钉死——未来实现者易误以为「澄清该上 quick-reply chips / 假设该 inline-edit」，引入 0010 已否决的 UI 原语 + 0005/0028 已否决的「解析 LLM 输出」脆弱性。本 ADR 收口该交互决策。

## Why

1. **守 0010「不引入额外 UI 原语」**——澄清 / 纠偏复用「下一条自然语言轮次」回路，与自然语言重定向同通道；chips / inline-edit 是 0010 已否决范式。
2. **守「不解析 LLM 输出」（0005/0028）**——quick-reply chips 须把 LLM 自由文本澄清**解析成选项**，把 0005 已否决的「解析 LLM 输出」脆弱性从 SQL 层搬到 UI 层。
3. **自由文本更表达**——用户可答「按地区，但只看华东」，不止选 A/B；chips 二选一堵死表达。
4. **assumption 是 LLM 输出非用户输入**——inline-edit 旁注文本语义错（改它不改 SQL 语义）；它是「读了纠偏」的抓手，非可编辑字段。
5. **纠偏长新枝非覆盖**——守 0010 分枝 + 0041 不复活；纠偏产新 `result_N`，原轮按 0028 恒可见留史。

## Considered options

- **澄清 quick-reply chips（点选 A/B）**：新 UI 原语（背叛 0010）+ 须解析 LLM 文本成选项（背叛 0005/0028）+ 表达力降。**否决**。
- **inline-edit assumption 文本**：assumption 是 LLM 输出，改它不动 SQL 语义；语义错。**否决**。
- **纠偏 = 原地重跑 / 覆盖原结果**：违反 0010 分枝 + 0041 不复活。**否决**。
- **QuestionBar 预填魔法内容**：预填假设了用户想说啥；澄清文本已在 workspace，用户读了自答即可。**否决**——最多占位提示。

## Consequences

- **澄清应答**：B 轮（0047 `○` 卡 + workspace 文本载荷）后，用户在 QuestionBar 输入下一条提问作答；无 chips / 选项 / 预填。
- **假设纠偏**：workspace 结果区的 `assumption` 旁注可点；点击聚焦 QuestionBar + 占位提示；纠偏作新轮、产新 `result_N`，原轮不动。
- **复用既有回路**：无新 UI 原语、无新 IPC、无新契约字段；澄清 / 纠偏仅是「下一条自然语言轮次」的特殊 framing。
- **累积成本**：纠偏会累积「误读 → 纠正」成对轮次（原轮 + 纠偏轮都恒可见，0028）——不可变 / 分枝模型的必然代价。缓解（**非新原语**，只是卡片元数据，同 0047 active chip）：纠偏轮卡可带 `纠偏 result_N` 反向 chip 让关系可见。该细节留视觉系统 ADR。
- **延伸 ADR-0010**：0010「点击历史步塞引用 + 不开新原语」延伸至「点 assumption 旁注聚焦 QuestionBar」——同一哲学的纠偏落地。
- **延伸 ADR-0018**：0018 澄清文本（B 轮）的**应答 UX** 定为「下一条自然语言轮次」（无 chips）。
- **延伸 ADR-0009**：`assumption` 字段的纠偏交互（点旁注 → 新轮）明确。
- **限定于 ADR-0045 shell**：澄清文本载荷 + assumption 旁注的渲染位置在 workspace（0045）；本 ADR 定交互，渲染位置归 0045 / 0047。
- **ADR-0028 仅引用非校准**：本 ADR 引用「B 轮占 N=20 窗口槽」服务澄清 + 回答的上下文连贯，但不改 0028 契约——故不加 0028 反向指针。
