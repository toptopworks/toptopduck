# Multi-source sessions: shared FROM namespace, monotonic result numbering, active-source = most-recently-uploaded

## Decision

v1 会话**允许多个源 Dataset 同时存在**。三条配套规则：

1. **起始当前表（Active Dataset）= 最近上传的源 Dataset**；跨源切换走**自然语言点名**（复用 ADR-0010 重定向回路），不设"切换当前表"按钮；仅当默认站不住（点名不存在的源 / 跨源键字段多重歧义）才升级澄清（窄门槛，ADR-0018）。
2. **全部 Dataset（源 + 中间结果）共享同一个 `FROM` 命名空间**，平等引用。
3. **中间结果编号会话级单调递增、永不复用**——删除（ADR-0013 soft/GC）后留空洞，下一个新结果取 `result_{max+1}`，不补坑。

源 Dataset 有机器名（默认文件名去扩展名 / Excel sheet 名），进同一命名空间；多源同名、用户重命名撞名由工具侧去冲突 / 拒绝（具体规整算法为 impl）。

## Context

ADR-0013「每轮 prompt 含**全部源 Dataset** 全 schema」与 ADR-0017「JOIN/UNION 列入 IN」都暗示多源；但 ADR-0010「会话开始时即源 Dataset」、ADR-0012「每个源 Dataset copy-in」是单数 / 不点名措辞，多源下「起始当前表是谁」「`result_N` 怎么不撞」未定。叠加 ADR-0010 的分支长新枝，命名空间与编号规则成为必须闭合的依赖。

## Why

1. **多源 IN 是对 ADR-0013/0017 的确认而非新增**：反悔多源 = 自相矛盾（0013 已承诺复数源进 LLM 工作集）；JOIN/UNION 在「单源 + 其派生」下几乎退化。
2. **最近源 = 起始当前表**是唯一可用且无歧义的隐式信号（刚上传的通常即想分析者），不需要用户额外操作；自然语言切换不戳破 ADR-0010 藏链心智（KISS）。
3. **编号永不复用**保住 ADR-0013 软失效的全部价值——复用会让历史视图里的「`result_3`」在不同时间指不同表，血缘断裂；空洞无害（stale 不进 LLM 工作集）。呼应 ADR-0012 源快照不可变哲学。
4. **共享 `FROM` 命名空间**是 CONTEXT「Dataset 是最小引用单元」的自然推论，无需分两个名字域。

## Considered options

- **v1 仅单源**：与 ADR-0013/0017 自相矛盾，且阉割 JOIN/UNION。**否决**。
- **多源但强制每次澄清当前表**：安全但打断非技术用户，违背 ADR-0018 窄门槛。**否决**。
- **复用被删编号（紧凑编号）**：破坏血缘 / 历史信任（ADR-0013 软失效意义归零）。**否决**。
- **新增"切换当前表"按钮**：戳破 ADR-0010 藏链心智，与定位相悖。**否决**。

## Consequences

- **校准 ADR-0010**：「会话开始时即源 Dataset」的单数措辞明确为「会话开始时即**最近上传的源 Dataset**」；多源共存时 LLM 候选源增多，"那个 / 上一步"指错概率上升——靠 ADR-0018 assumption 标注 + ADR-0010 历史视图标签纠偏（既有回路，多源只是更频繁触发，不引入新机制）。
- **校准 ADR-0012**：「每个源 Dataset copy-in」明确含多源；源 Dataset 在会话 DB 里是有机器名的表，与 `result_N` 同域。
- **CONTEXT 已更新**：工作集定义为"一个或多个源 Dataset"；当前表定义为"会话开始时即最近上传的源"。
- 编号无上界：M=100（ADR-0013）只管"同时存在的 `result_N` 数量"，不卡编号；长会话下 LLM 可能面对 `FROM result_N`（N≫100），窗口外靠 ADR-0013 摘要兜底，机器名无需对用户有意义。
- 多源去冲突 / 重命名全局唯一为工具侧 impl，不阻塞架构决策。
