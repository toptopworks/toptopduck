# Desktop application, local-first

## Decision

toptopduck 以**本地优先的桌面应用**形态交付：数据集与执行都留在用户本机，只有 LLM 推理调用会发往云端模型 API。

## Context

toptopduck 是 AI 数据分析工具，用户上传私密数据集（Excel/CSV/JSON/Parquet）并用自然语言操作。数据隐私是这类工具的命门。

## Why

"数据不出本机"是核心卖点，并直接消解"在共享环境里执行生成代码"的云端安全负担。"日常数据分析"场景天然是单用户、本地文件导向，而非多租户协作平台。

## Considered options

- **本地 Web 应用**（localhost server + 浏览器 UI）：隐私同等，但需用户手动起服务，体验不如可安装 App。
- **多租户 Web SaaS**：可触达更广用户、可协作；但需扛认证 + 租户隔离 + 云端执行沙箱 + 对象存储，基础设施与安全成本高一个量级。**否决**。
- **纯 CLI / 库**：违背"用自然语言服务非技术用户"的定位。**否决**。

## Consequences

- 需打包并分发桌面运行时，更新链路成为持续关注点。
- **明确不做**：多用户协作、浏览器随处访问（至少本期不在范围内）。
- 仅 LLM 调用联网——隐私模型成立的前提是用户接受数据集本身不上云。
- **v1 目标用户须具备 Claude 可达的网络接入**（见 ADR-0019）：大陆纯小白市场不在 v1 范围，网络门槛作为诚实准入披露。
