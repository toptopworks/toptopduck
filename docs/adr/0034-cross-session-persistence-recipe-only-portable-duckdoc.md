# Cross-session persistence in v1: longevity-local, recipe-only rebuild (not snapshot), portable user-owned .duck document, eager replay, per-turn atomic write

## Decision

1. **持久化进 v1 scope——"本地优先"读法为 longevity-local**：数据与分析持久存活在本地、重启可恢复。**取代** ADR-0027/0012 的"纯内存 / 关即重置"框定（保留它们的运行态内存隔离、单飞行、derive-only 保证）。CONTEXT「会话」定义相应从"关闭即重置"改为持久化单位。

2. **持久化单位 = 配方（recipe），非数据快照，按当前态组织（非历史账本）**。recipe 描述**当前工作集**，分两部分：**可重建部分** = 当前源集（路径 + 规整参数 + 内容指纹）+ 当前仍有效的 productive SQL 链（ADR-0009，被换源级联失效的轮**不在此列**，ADR-0025）；**展示部分** = 全量对话历史（所有轮恒可见，ADR-0028，纯追加永不裁剪）。外加 `format_version`、session 名、active dataset 指针。**本身不含物化数据**（ADR-0004 derive-only）：每个中间结果是 source + SQL 的确定性函数，故 recipe 是当前工作集的**完整无损**描述。缓存落盘是优化、非真相来源，v1 不做（YAGNI）。**源生命周期操作**（加源 ADR-0022 / 换源 ADR-0025 / 删源）原子落 recipe：改当前源集 + **裁剪**可重放链中失效轮 + **追加**对话历史一条可见事件。换源后旧源可能不可得 → 失效轮无法重算 → 故按当前态组织、**从不重算死轮**（否决全量操作账本）。

3. **形式 = 可移植用户文档（.duck）**。一个 Session = 一个用户可见、可移动的 recipe 文件；**源数据文件独立留在磁盘、被路径引用，不进 .duck**。厘清：`.duck` 是 recipe **文本文档**，**非** DuckDB 持久库文件——0012 否决的"copy-in 到持久 .duckdb"是**数据快照**方案，与本 recipe 方案不同物。

4. **resume = eager 整链重放**。打开 .duck → 读 recipe → 读源 + 完整性校验（见 ADR-0035）→ 校验通过后**按序重跑可重放链**（当前仍有效的 productive 轮——产出 `result_N` 的轮，ADR-0009；被换源级联失效的轮已在 recipe 中剔除，ADR-0025）；ADR-0028 的 no-result 轮（消歧 / 拒绝 / 失败 / 取消）**按历史静态呈现、不重跑**（消歧选择已记在 recipe）。配可见进度；某轮断裂则停在该轮、断点前全部物化（ADR-0028/0030 呈现）。

5. **写盘 = 每轮终态自动追加 + 原子写**。一轮达终态 outcome（产出 / 消歧 / 拒绝 / 失败 / 取消，ADR-0028）即**立即**把该轮写进 recipe；**在飞期间不写**（中途崩了丢弃该轮，诚实可接受——它没产出任何 durable 东西）。原子写（临时文件 + rename）防写崩 corruption；recipe 是小文本，整文件重写够（KISS），journaling 推迟（YAGNI）。**崩溃窗口 = 当前在飞那一轮**。显式"另存为 / 导出副本"保留为非持久功能（复制 .duck），不是持久化路径。

## Context

ADR-0012（纯内存会话）、0027（多会话内存隔离）都把会话框成临时、关即重置；0027 甚至在 Considered options 明确否决"会话持久化"为 v1 不做，0012 亦否决"copy-in 到持久 .duckdb 文件（会话恢复）"。但 CONTEXT 第一句"本地优先"在熟悉 local-first 运动的读者心中默认含"持久存活 / 离线可恢复"——全内存、关即清零对一个数据分析工具是反预期硬伤（做一晚上链式分析、关掉全没）。是 longevity 承诺下的未闭合依赖。

## Why

1. **longevity-local 是"本地优先"的正确读法**：隐私本地（数据不离机）只是底线；熟悉该运动的用户/工程师默认期待"离线可恢复、不丢工作"。读法 A 会让"优先"落空成"只是不上云"。
2. **recipe-only 完整无损且近乎零成本**：0004（derive-only）+ 0003（materialize-each-step）已让每个中间结果成为 source + SQL 的确定性函数——recipe 是工作集的完整描述是"零成本继承现有架构"，非新设计；存派生数据则是重复存储可重算之物。
3. **可移植文档兑现"用户以真文件拥有数据"**：隐藏 app 托管存储重新制造"我的数据去哪了"的不透明，正是 local-first 要消灭的。recipe 是纯文本，可移植性在这里白送（不像快照要背 GB），并解锁双击打开 / 最近文件 / 分享 / git 版本控制。
4. **eager 整链重放状态模型极简 + 诚实前置**：工作集要么"整链物化完成"、要么"在某轮断裂、断点前都已物化"——一个确定状态；lazy 引入部分物化 + 依赖 DAG 传递物化的状态机，且把失败散落到用户每次触碰，不诚实。eager 延迟随轮数增长但配可见进度即诚实可见；对数据分析工具重开会话等几秒符合常识。
5. **每轮终态原子写兑现"永不丢工作"，崩溃窗口最小**：关即写 / 显式保存重新引入"崩即重置 / 丢工作"，背叛 longevity；只有每轮终态写能做到"崩溃最多丢在飞那轮"。原子写堵住"写盘中崩 → 损坏整文件（比丢一轮更糟）"。

## Considered options

- **数据快照持久化（存物化表）**：GB 级存储 + 快照版本管理 + 跨 app 版本 schema 迁移 + 快照与内存态一致性，整套子系统对 v1 不成比例；且派生数据可由 source + SQL 重算。**否决**。
- **不透明 app 托管存储（隐藏目录 / 内部 SQLite）**：重新制造"我的数据去哪了"不透明，背叛 local-first"用户拥有数据"。**否决**。
- **lazy 按需物化**：部分物化 + 依赖 DAG 传递物化的状态机，复杂度不成比例；失败散落、不诚实。**否决（v1）**。
- **关闭时写 / 显式保存**：重新引入"崩即重置 / 丢工作"，背叛 longevity。**否决**。
- **recipe + 缓存混合（缓存作真相）**：缓存一旦是真相来源便同快照方案要管一致性；作为优化可后加，v1 不做。**否决（v1）**。

## Consequences

- **校准 ADR-0027**：0027 的 Considered options 中「会话持久化（多会话+可恢复）…v1 不做。否决（v1）」被反转——会话现为持久化单位（recipe-only、可移植 .duck、eager 重放、每轮原子写）。保留：多会话**运行态**内存隔离、各自单飞行（0021）、各自 `memory_limit`（0005）——是打开态语义，持久化不改变它们。0027 Consequences 已追加校准指针。
- **校准 ADR-0012**：源 copy-in 不可变快照仍是**运行态**形态（resume 时按 recipe 重读源文件重建该快照）；本 ADR「v1 不做会话持久化 / 关即重置」被反转；copy-in 冻结、源只读、类型推断不可逆（0004）等保证不变——resume 只是重新创建运行态快照。0012 Consequences 已追加校准指针。
- **校准 ADR-0029**：本 ADR 不变量 2「默认零用户数据持久落盘」精确化——持久 at-rest 现在存在，但**仅限用户拥有的 recipe（.duck）：本地、用户可见、无密钥**（见 ADR-0036 secrets-never）；本 ADR 真实隐私保证（对作者零外发、无隐藏落盘、key-in-Rust）全部保留。0029 Consequences 已追加校准指针。
- **CONTEXT 已更新**：会话重定义为持久化单位（删"关闭即重置"）；新增「配方（Recipe）」术语（含内容指纹；按当前态组织，非历史账本）。
- **与 ADR-0025 / 0022 的交互**：换源（0025）级联失效的 productive 轮在落 recipe 时即从可重放链剔除（保留于对话历史展示）；加源（0022）追加进当前源集；删源同理裁剪。resume 只重放当前有效的可重放链，**从不重算被换源失效的死轮**（旧源可能不可得）。源生命周期操作是唯一裁剪可重放链的情形——对话历史恒为纯追加。
- **不变量：resume 离线**。重新打开 .duck 全程本地（读源 + 完整性校验 + eager SQL 重放 + 渲染历史），**不需云 LLM**——SQL 与消歧选择已存 recipe、直接执行；viz 不在 resume 时重算（重开结果以表格呈现，见 ADR-0036）。**只有创作新轮才需联网**（ADR-0006）。是 local-first 的强保证：保存的分析断网可打开。
- 实现侧：.duck 读写、resume 重放管线、每轮终态原子追加、源路径解析（见 0036）。
- 缓存加速、lazy 物化、journaling 均明确 v1 不做，留待延迟真正成问题时另开 ADR。
- **被 ADR-0037 校准**：recipe 的 productive SQL 链、active-dataset 指针均存**引用名**（创建时确定、恒定，不受重命名影响）；「中间结果名 / 重命名」（ADR-0036 内容边界）= 引用名 + 显示别名（可重命名）。重命名只动显示层、不波及可重放链 → resume 重放自洽，见 ADR-0037。
